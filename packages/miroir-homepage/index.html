What is Miroir?

Inspired by Smalltalk's interactive model, Miroir eliminates the wall between design‑time and runtime. Every concept — entities, queries, transformers, reports — lives as a first‑class, executable JSON object in the running application.

Schema‑First

Define your data model as Jzod schemas (JSON + Zod). UI forms and runtime validations are generated automatically — zero duplication, zero drift.

Live Environment

Modify entities, queries and reports while the app is running. No compile–wait–restart cycle. Instant feedback, just like Smalltalk’s legendary interactive model.

Multi‑Backend

One codebase, multiple persistence backends: PostgreSQL, MongoDB, filesystem, IndexedDB, and more. Switch with a configuration change — no code changes needed.

AI‑Ready

First‑class MCP (Model Context Protocol) support. LLMs can inspect, query, and modify your application model at runtime through a well‑defined, structured interface.

Composable Queries

A layered query language: extractors fetch data, combiners join it, transformers shape the result. The same declaration runs in‑memory, in the browser, or as SQL.

Open Source · MIT

Fully open, self‑hosted, and auditable. Run on your own infrastructure, contribute to the core, or fork and adapt for your domain needs — no vendor lock‑in.

Core Concepts

Everything in Miroir is a first‑class, versioned, introspectable object living in the running system.

Entity

A typed, schema‑validated data class. Single, composite and UUID primary keys; relations all first‑class.

Query

Composable read operations: extractors fetch raw data, combiners join it, transformers shape the result.

Transformer

Pure functions on data, declared as JSON. The same declaration drives in‑memory, server‑side, or SQL execution.

Report

Declarative UI pages that bind a query to a layout. Tables, forms, and custom sections — configurable at runtime.
